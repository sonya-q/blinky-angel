# Privacy Policy - Blinky Angel

**Last updated:** March 2026
Blinky Angel is a Chrome browser extension designed to support users’ digital eye health by monitoring blink rate locally and providing break reminders.

This Privacy Policy explains how the extension handles user data.

## 1. Information We Collect

Blinky Angel **does not collect, transmit, store, or share any personal data.**

The extension may access the following information **locally on the user’s device only**:

- Webcam video stream (used solely for real-time blink detection)
- Blink statistics such as:
  - total blink count
  - blink rate per minute
- User preferences such as reminder sensitivity settings and interface toggles

All such information is processed entirely within the user’s browser and **never leaves the user’s device**.

## 2. How Information Is Used

Local data is used only to provide the extension’s core functionality, including:

- Real-time blink detection and eye-health monitoring
- Displaying reminder notifications (such as the 20-20-20 rule)
- Maintaining local statistics and personalized reminder settings
- Improving the user’s on-screen experience with the reminder interface

No analytics, profiling, tracking, or behavioral monitoring is performed.

## 3. Data Storage

Any statistics or settings generated by Blinky Angel are stored **locally using the browser’s storage APIs**.

This data:

- Is not uploaded to any server
- Is not accessible by the developer
- Can be cleared by the user at any time by removing the extension or clearing browser storage

## 4. Webcam Access

Blinky Angel requests camera permission only to enable blink detection functionality.

- Camera access occurs **only after explicit user consent**
- Video data is processed **in real time and is never recorded, saved, or transmitted**
- Users can revoke camera permission at any time through browser settings

## 5. Data Sharing

Blinky Angel **does not sell, share, transfer, or disclose any user data** to:

- Third-party companies
- Advertising networks
- Analytics providers
- Cloud services
- Government agencies (unless legally required)

All processing occurs locally in the browser.

## 6. Third-Party Libraries

The extension uses trusted open-source computer vision libraries (such as MediaPipe Face Mesh and TensorFlow.js) loaded from a public content delivery network for performance purposes.

These libraries are used solely for **on-device facial landmark detection** and do not transmit user data externally.

## 7. User Control

Users maintain full control over the extension, including:

- Granting or revoking camera permissions
- Clearing local extension storage
- Disabling or uninstalling the extension at any time 8.

This Privacy Policy may be updated from time to time.
Any updates will be reflected by a revised “Last updated” date.
